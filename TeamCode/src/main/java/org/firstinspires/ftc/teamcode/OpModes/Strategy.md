# DECODE 机器人操作代码说明

## 1. 自动阶段算法

### 1.1 自动阶段概述

当前 DECODE 程序主要为 TeleOp（手动控制）模式，自动阶段（Autonomous）算法需要单独实现。以下是基于现有代码结构设计的自动阶段算法：

### 1.2 自动阶段算法设计

#### 1.2.1 初始位置设置

- **红色队伍**：从红色联盟区开始，初始位置设置为 `RedResetPose`
- **蓝色队伍**：从蓝色联盟区开始，初始位置设置为 `BlueResetPose`

#### 1.2.2 自动任务流程

1. **初始化阶段**：
   - 加载 DRL 模型（如果可用）
   - 初始化所有硬件组件
   - 设定初始位置和朝向
2. **自动导航**：
   - 使用 RoadRunner 库进行路径规划
   - 导航到预设的射击位置（大三角或小三角区域）
   - 自动调整朝向以对准目标
3. **自动射击**：
   - 根据距离计算发射速度（使用传统方法或 DRL）
   - 等待发射速度达到目标值
   - 自动发射预设数量的弹丸
4. **自动收集**：
   - 导航到弹丸收集区域
   - 启动清扫器收集弹丸
   - 再次导航到射击位置进行二次射击
5. **结束阶段**：
   - 导航到指定的结束位置
   - 停止所有电机和执行器

        COLLECT,
        NAVIGATE_TO_END,
        DONE

    private AutoState currentState = AutoState.INIT;
    
    @Override
    public void runOpMode() {
        // 初始化硬件
        Init();
        
        waitForStart();
        
        while (opModeIsActive()) {

                    break;

                        currentState = AutoState.SHOOT;

                    break;
                case NAVIGATE_TO_COLLECT:
                    // 导航到收集位置
                    break;
                case COLLECT:
                    // 收集弹丸
                    break;
                case NAVIGATE_TO_END:

        }
    }
}

## 2. 手动阶段操作逻辑

### 2.1 基本控制

#### 2.1.1 底盘控制

- **游戏手柄**：Gamepad1（一操）和 Gamepad2（二操）
- **控制方式**：
  - **前进/后退**：左摇杆 Y 轴（一操为主，二操为辅）
  - **左右平移**：左摇杆 X 轴（一操为主，二操为辅）
  - **旋转**：右摇杆 X 轴（一操为主，二操为辅）
  - **无头模式切换**：Gamepad1 X 按钮（释放时切换）

#### 2.1.2 射击控制

- **进入射击模式**：Gamepad1 Y 按钮（切换 ReadyToShoot 状态）
- **自动对准**：Gamepad1 左扳机（按下超过 0.6）
- **手动触发**：Gamepad2 Start 按钮（在射击模式下控制清扫器和触发器）

#### 2.1.3 清扫器控制

- **进入吃球模式**：Gamepad1 左保险杠或 Gamepad2 左保险杠
- **进入吐球模式**：Gamepad1 右保险杠或 Gamepad2 右保险杠
- **停止清扫器**：切换到等待状态

#### 2.1.4 攀爬控制

- **进入攀爬模式**：Gamepad1 B 按钮
- **电梯控制**：Gamepad2 左右扳机（直接控制）或左右摇杆按钮（导航到攀爬点）

#### 2.1.5 DRL 模式切换

- **切换 DRL 模式**：Gamepad2 B 按钮
- **模式顺序**：RunDRL →NoDRL → FullDRL → RunDRL 

### 2.2 机器人状态

#### 2.2.1 状态枚举

```java
public enum ROBOT_STATUS {
    EATING,      // 吃球状态
    WAITING,     // 等待状态
    OUTPUTTING,  // 吐球状态
    SHOOTING,    // 射击状态
    EMERGENCY_STOP,  // 紧急停止状态
    CLIMBING     // 攀爬状态
}
```

#### 2.2.2 状态转换

- **EATING → WAITING**：按下 Gamepad2 X 按钮
- **WAITING → EATING**：按下 Gamepad1 左保险杠或 Gamepad2 左保险杠
- **WAITING → OUTPUTTING**：按下 Gamepad1 右保险杠或 Gamepad2 右保险杠
- **OUTPUTTING → WAITING**：再次按下 Gamepad1 右保险杠或 Gamepad2 右保险杠
- **WAITING → SHOOTING**：按下 Gamepad2 Y 按钮或 ReadyToShoot 为 true 且航向误差小于阈值
- **任何状态 → WAITING**：按下 Gamepad1 A 按钮或 Gamepad2 A 按钮（非攀爬模式）
- **任何状态 → CLIMBING**：按下 Gamepad1 B 按钮
- **CLIMBING → WAITING**：再次按下 Gamepad1 B 按钮

### 2.3 特殊功能

#### 2.3.1 自动导航

- **导航到近侧射击点**：Gamepad1 下方向键
- **导航到远侧射击点**：Gamepad1 上方向键
- **导航到攀爬点**：攀爬模式下按下 Gamepad2 左右摇杆按钮

#### 2.3.2 参数调整

- **调整航向误差阈值**：Gamepad1 右方向键（增加）
- **调整角度偏移**：Gamepad2 左右方向键
- **调整速度修正系数**：Gamepad2 上下方向键

#### 2.3.3 DRL 功能

- **FullDRL**：始终使用 DRL 计算发射速度
- **RunDRL**：当机器人速度大于 0.1 m/s 时使用 DRL，否则使用传统方法
- **NoDRL**：始终使用传统方法计算发射速度

## 3. 战术设计

### 3.1 基本战术

#### 3.1.1 红色队伍战术

1. **开局**：从红色联盟区出发，导航到小三角区域
2. **第一波射击**：对准目标发射所有预加载的弹丸
3. **收集弹丸**：导航到弹丸收集区，收集地面上的弹丸
4. **第二波射击**：再次导航到射击位置，发射收集到的弹丸
5. **攀爬**：最后导航到攀爬点，完成攀爬动作

#### 3.1.2 蓝色队伍战术

1. **开局**：从蓝色联盟区出发，导航到小三角区域
2. **第一波射击**：对准目标发射所有预加载的弹丸
3. **收集弹丸**：导航到弹丸收集区，收集地面上的弹丸
4. **第二波射击**：再次导航到射击位置，发射收集到的弹丸
5. **攀爬**：最后导航到攀爬点，完成攀爬动作

### 3.2 高级战术

#### 3.2.1 动态射击策略

- **近程射击**：使用较低的发射速度（speed25\_55）
- **远程射击**：使用较高的发射速度（speed35\_55）
- **移动射击**：使用 RunDRL 模式，在移动中进行射击

#### 3.2.2 弹丸管理

- **优先收集**：优先收集靠近机器人的弹丸
- **批量射击**：收集一定数量的弹丸后再进行射击，提高射击效率
- **应急方案**：当弹丸不足时，切换到精确射击模式，提高命中率

#### 3.2.3 防守策略

- **区域控制**：通过占据关键位置，阻止对手收集弹丸
- **干扰对手**：在对手射击时进行干扰，使其命中率降低
- **快速响应**：使用自动导航功能快速切换位置，应对对手的战术变化

### 3.3 战术执行要点

#### 3.3.1 操作技巧

- **熟练掌握无头模式**：在复杂环境中使用无头模式可以更灵活地控制机器人
- **合理使用自动对准**：在射击前使用自动对准功能，提高射击精度
- **及时切换 DRL 模式**：根据机器人速度和环境情况，选择合适的 DRL 模式

#### 3.3.2 团队配合

- **一操负责**：机器人的整体移动和射击控制
- **二操负责**：弹丸管理和参数调整
- **沟通协调**：在比赛中保持良好的沟通，及时调整战术

#### 3.3.3 应急处理

- **卡弹处理**：当清扫器卡弹时，立即切换到吐球模式
- **发射失败**：当发射失败时，检查发射速度和航向，调整后重试
- **硬件故障**：当硬件出现故障时，立即切换到等待状态，尝试修复或调整战术
