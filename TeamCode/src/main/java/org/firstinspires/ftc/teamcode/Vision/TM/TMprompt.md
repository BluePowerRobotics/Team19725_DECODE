请用 Java 和 EasyOpenCV在TmpMatcher.java文件中编写一个实时摄像头目标检测与跟踪程序，实现TmpMatcher类满足以下要求：

## 整体功能

1. 实例化TmpMatcher对象时从template文件夹加载指定名称的模板图像作为视觉基准。
2. 从默认摄像头实时读取帧。
3. 对每一帧进行**多目标检测与跟踪**，实现：
   - 检测不同大小、不同旋转角度的小球（利用特征点匹配的尺度/旋转不变性）
   - 同时跟踪多个小球
   - 输出每个对象在图像中的位置（中心或左上角）和大小（宽度、高度）
   - 对遮挡和高速运动具有鲁棒性（通过卡尔曼滤波和跟踪机制）

## 技术方案（必须包含以下要点）

### 1. 特征点匹配（用于检测）

- 使用 **SIFT** 特征（若环境不支持，可降级为 ORB，但优先 SIFT）提取模板和每一帧的关键点与描述子。
- 使用 `BFMatcher` 进行 `knnMatch`（k=2），应用 Lowe’s 比率测试（ratio=0.75）筛选优质匹配。
- 对匹配点进行空间聚类（例如基于欧氏距离的 DBSCAN，阈值 30 像素），每个聚类对应一个候选对象。
- 对每个有效聚类（至少 4 个匹配点），通过 `findHomography`（RANSAC）计算单应性矩阵，将模板角点投影到当前帧，得到包围四边形，进而计算最小外接矩形，获得位置和大小。

### 2. 目标跟踪（提升鲁棒性）

- 为每个已检测到的对象维护一个**跟踪对象**（TrackedObject），包含：
  - 当前帧中的位置（矩形）
  - 卡尔曼滤波器（用于预测下一帧位置）
  - 连续丢失帧计数器（如果连续多帧未匹配到，则删除该对象）
- 在每一帧中，首先利用**特征点匹配**进行检测，得到当前帧的新检测结果列表。
- 使用**匈牙利算法**或简单的**IOU匹配**将新检测结果与已有的跟踪对象进行关联：
  - 若关联成功，更新跟踪对象的位置、卡尔曼滤波器，并将丢失计数器置零。
  - 若未关联的跟踪对象，则使用卡尔曼预测的位置作为当前帧位置，并增加丢失计数器；若丢失超过阈值（如 5 帧），则删除该对象。
  - 若未关联的新检测结果，则创建新的跟踪对象（并为其初始化卡尔曼滤波器）。

### 3. 卡尔曼滤波（处理遮挡与高速运动）

- 每个跟踪对象使用**卡尔曼滤波器**（OpenCV 中可使用 `KalmanFilter` 类）对运动状态进行建模。
- 状态向量：`[x, y, vx, vy, w, h]` 或 `[x, y, vx, vy]`（大小可假设短期不变），根据实际需求设计。
- 测量向量为当前帧检测到的位置（x, y）。
- 当检测丢失时，使用卡尔曼预测值作为当前帧显示位置，保持跟踪持续。

### 4. 多目标非极大值抑制

- 对所有检测结果（包括跟踪器预测和检测器输出）执行 NMS（IoU 阈值 0.3），避免同一目标重复绘制。

### 5. 实时性能优化

- 可降低图像分辨率（例如 640×480）以提升处理速度。
- 限制 SIFT 特征点数量（如 `SIFT.create(1000)`）。
- 在跟踪阶段，可仅在部分帧（如每 3 帧）运行完整的特征点检测，其余帧仅使用跟踪器进行位置更新（但为了简单，可每帧都做检测，但需权衡）。

## 代码结构要求

- 定义一个 `TrackedObject` 类，封装卡尔曼滤波器、矩形框、丢失计数等。
- 定义一个 `Detector` 类，封装模板加载、特征点匹配、单应性估计等检测逻辑。
- 使用 `Mat` 对象时注意释放（`.release()`），避免内存泄漏。
- 处理摄像头打开失败、模板加载失败等异常情况。
- 实时显示标注后的画面（矩形框、ID、大小），按 'q' 退出。

## 附加说明

- 若 SIFT 不可用（缺少 contrib 模块），可改用 ORB，并在注释中注明。
- 聚类算法可简单实现为基于距离的邻域聚类（类似 DBSCAN），或使用 OpenCV 的 `partition` 函数进行连通组件分组。
- 匈牙利算法可用简单贪心匹配代替（根据 IOU 排序），但需确保多目标正确关联。
